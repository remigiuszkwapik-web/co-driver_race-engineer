# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- **Dev server:** `bun dev`
- **Build:** `bun run build`
- **Preview production build:** `bun run preview`
- **Lint:** `bun run lint`
- **Typecheck:** `bun run typecheck`
- **All tests (unit + nuxt):** `bun test`
- **Unit tests only:** `bun test:unit`
- **Nuxt component tests only:** `bun test:nuxt`
- **Single test file:** `bun vitest run test/unit/example.test.ts`
- **E2E tests (Playwright):** `bun test:e2e`
- **E2E with UI:** `bun test:e2e:ui`

Package manager is **bun**. Do not use npm, yarn, or pnpm.

## Architecture

This is a **Nuxt 4** app using the Nuxt 4 directory structure (`app/` directory for client code). Key modules:

- **@nuxt/ui** (v4) — component library with Tailwind CSS v4. Components are auto-imported with `U` prefix (e.g., `UButton`, `UPageHero`).
- **@nuxthub/core** — NuxtHub integration for edge deployment
- **@nuxt/scripts** — third-party script management
- **@vueuse/nuxt** — Vue composables, auto-imported
- **drizzle-orm** + **@libsql/client** — database layer (LibSQL/Turso)
- **@nuxt/eslint** — ESLint config generated from Nuxt, extended in `eslint.config.mjs`

### Directory layout

- `app/` — client-side code: pages, components, assets, `app.vue`, `app.config.ts`
- `server/` — server routes and API (Nitro) — currently empty, ready for use
- `test/unit/` — pure unit tests (node environment, vitest)
- `test/nuxt/` — component tests with Nuxt environment (uses `mountSuspended` from `@nuxt/test-utils/runtime`)
- `tests/` — Playwright E2E tests (uses `@nuxt/test-utils/playwright`)

### Testing setup

Vitest is configured with two projects in `vitest.config.ts`:
- **unit** — `test/unit/*.{test,spec}.ts`, plain node environment
- **nuxt** — `test/nuxt/*.{test,spec}.ts`, Nuxt environment with happy-dom

Playwright E2E tests run against Chromium by default.

### Style conventions

ESLint enforces stylistic rules: no trailing comma (`commaDangle: 'never'`), 1TBS brace style. The ESLint config is generated by Nuxt — customize via `nuxt.config.ts` `eslint` key, not directly in `eslint.config.mjs`.

UI theme colors: primary=green, neutral=slate (set in `app/app.config.ts`).

Icon sets available: `lucide` and `simple-icons` (via `@iconify-json/*`).

## Database

- **Database Dialect**: The database dialect is set in the `nuxt.config.ts` file, within the `hub.db` option or `hub.db.dialect` property.
- **Drizzle Config**: Don't generate the `drizzle.config.ts` file manually, it is generated automatically by NuxtHub.
- **Generate Migrations**: Use `npx nuxt db generate` to automatically generate database migrations from schema changes
- **Never Write Manual Migrations**: Do not manually create SQL migration files in the `server/db/migrations/` directory
- **Workflow**:
  1. Create or modify the database schema in `server/db/schema.ts` or any other schema file in the `server/db/schema/` directory
  2. Run `npx nuxt db generate` to generate the migration
  3. Run `npx nuxt db migrate` to apply the migration to the database, or run `npx nuxt dev` to apply the migration during development
- **Access the database**: Use the `db` instance from `@nuxthub/db` (or `hub:db` for backwards compatibility) to query the database, it is a Drizzle ORM instance.
